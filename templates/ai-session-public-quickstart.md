# AI 会话公开化快速清单

这份清单用于把一次 AI 会话整理成可公开、可验证、可复用的作品。适合网页、脚本、文档、图片包、表格或小工具。

## 适用场景

- 你已经用 AI 做出一段有价值的结果。
- 你希望别人能打开、检查或复用它。
- 你不想公开 token、账号、本地路径、私密聊天或未验证承诺。

## 六步收尾

1. 定一个入口
   写清楚作品名称、公开 URL、本地运行方式和目标读者。

2. 划公开边界
   列出不能公开的内容：密钥、账号、私信、客户信息、本机路径、原始聊天记录。

3. 留最小证据
   至少保留 README、验证命令、公开链接、关键截图或机器可读数据。

4. 跑真实验证
   检查本地命令、公开 HTTP 状态、桌面布局、移动布局和敏感信息扫描。

5. 做传播出口
   准备一个标题、一张图、一段短说明、一份证据包和一个反馈入口。

6. 回写问题
   发布后的评论、误解、链接问题和新需求，要回到 issue、日志或下一版清单。

## 最小文件包

- `README.md`：作品是什么，怎么打开，怎么验证。
- `evidence-pack.md`：公开链接、验证命令、边界说明。
- `public-health.json`：当前公开状态和检查项。
- `materials-index.json`：公开材料索引。
- `templates/`：可复用模板或清单。

## 发布前自检

- 公开链接能打开。
- 验证命令能在本地跑完。
- 没有 token、API key、账号 ID、本机绝对路径。
- 页面在手机和桌面都能读。
- README 里没有只对自己有用的暗号。
- issue 或日志里记录了为什么做、怎么验证、下一步是什么。

## 发布后回流

- 把有效评论整理成问题，不搬运私人身份信息。
- 链接失效时修复并重新验证。
- 读者误解时改公开文案，不改事实。
- 新需求有复用价值时，写成新的模板或清单。
